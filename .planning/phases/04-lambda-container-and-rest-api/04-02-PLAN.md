  <files>lambda/test_local.sh</files>
  <action>
Create `lambda/test_local.sh` — a bash script that builds and tests the container locally.

The script should:

1. **Pre-flight checks:**
   - Verify `lambda/model_artifacts/model.onnx` exists (exit with clear error if not: "model_artifacts/ not found. Run Phase 3 ONNX export first, then copy model.onnx and tokenizer files to lambda/model_artifacts/")
   - Verify `lambda/model_artifacts/tokenizer_config.json` exists (exit with error if tokenizer files missing)
   - Verify Docker is running (`docker info` or similar)

2. **Build the image:**
   ```bash
   docker build --platform linux/amd64 --provenance=false \
     -t housing-predictor:local \
     lambda/
   ```
   Exit on build failure.

3. **Verify no PyTorch in image:**
   ```bash
   docker run --rm housing-predictor:local pip show torch 2>&1 | grep -q "not found"
   ```
   Print "PASS: No PyTorch in container" or "FAIL: PyTorch found in container" and exit.

4. **Check image size:**
   ```bash
   docker image ls housing-predictor:local --format "{{.Size}}"
   ```
   Print the size. Warn if > 3GB.

5. **Test with RIE (network enabled):**
   - Start container in background: `docker run --rm -d -p 9000:8080 --name hp-test housing-predictor:local`
   - Wait 5 seconds for cold start
   - Send a valid API Gateway v2 proxy event via curl:
     ```bash
     curl -s -XPOST "http://localhost:9000/2015-03-31/functions/function/invocations" \
       -H "Content-Type: application/json" \
       -d '{
         "version": "2.0",
         "routeKey": "POST /predict",
         "rawPath": "/predict",
         "headers": {"content-type": "application/json"},
         "requestContext": {"http": {"method": "POST", "path": "/predict"}},
         "body": "{\"bedrooms\":3,\"bathrooms\":2.0,\"sqft\":1800,\"lot_size\":0.25,\"year_built\":1995,\"zip_code\":\"07030\",\"property_type\":\"Single Family\"}",
         "isBase64Encoded": false
       }'
     ```
   - Print the response
   - Check that response contains "predicted_price" (basic sanity)
   - Stop and remove the container: `docker stop hp-test`

6. **Test with --network none (offline validation):**
   - Start container with `--network none`: `docker run --rm -d -p 9001:8080 --network none --name hp-test-offline housing-predictor:local`
   - Wait 5 seconds
   - NOTE: `--network none` means we CANNOT curl from host to container. Instead, use `docker exec` to run a Python test inside the container:
     ```bash
     docker exec hp-test-offline python -c "
     from transformers import AutoTokenizer
     tok = AutoTokenizer.from_pretrained('/var/task/model_artifacts')
     print('Tokenizer loaded offline OK:', tok.__class__.__name__)
     "
     ```
   - If this succeeds, tokenizer is confirmed bundled (not fetched from Hub)
   - Stop and remove: `docker stop hp-test-offline`

7. **Print summary:**
   - Build: PASS/FAIL
   - No PyTorch: PASS/FAIL
   - Image size: {size}
   - RIE prediction test: PASS/FAIL (with response snippet)
   - Offline tokenizer test: PASS/FAIL

Make the script executable: `chmod +x lambda/test_local.sh`

Set `#!/usr/bin/env bash` and `set -euo pipefail` at top.
