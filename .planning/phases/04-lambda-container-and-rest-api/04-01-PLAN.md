- `PredictResponse(BaseModel)`: predicted_price (float), predicted_price_rounded (int)

**POST /predict endpoint:**
- Call `format_prompt()` with all 7 fields from the request
- Tokenize the prompt with `tokenizer(prompt, return_tensors="np")`
- Implement autoregressive generation loop (NOT single-token):
  - Set `MAX_NEW_TOKENS = 12` (prices are at most ~7 digits)
  - Loop: call `session.run(None, feed)` where feed has `input_ids` and optionally `attention_mask`
  - Take `argmax` of logits `[0, -1, :]` for next token
  - Append to `generated_ids` list and concatenate to `input_ids` for next iteration
  - Break on `eos_token_id` or max tokens reached
  - Decode `generated_ids` with `tokenizer.decode(generated_ids, skip_special_tokens=True)`
- Call `parse_price_from_output(generated_text)` to extract the price
- If price is None or <= 0, raise `HTTPException(status_code=500, detail=...)` with the raw generated text
- Round price: `round(price / 1000) * 1000`
- Return `PredictResponse(predicted_price=price, predicted_price_rounded=rounded)`

**Mangum handler:**
- `handler = Mangum(app, lifespan="off")` — this is the Lambda entry point

**CRITICAL — do NOT:**
- Reimplement `format_prompt` or `parse_price_from_output` inline — import from `prompt_utils`
- Create `InferenceSession` inside the endpoint function — it MUST be module-level
- Use `from lambda.prompt_utils import ...` — `lambda` is a Python reserved keyword

```dockerfile
FROM public.ecr.aws/lambda/python:3.12

# Prevent HuggingFace Hub network calls — tokenizer and model must be bundled
ENV TRANSFORMERS_OFFLINE=1
ENV HF_DATASETS_OFFLINE=1
ENV TOKENIZERS_PARALLELISM=false

# Install inference-only dependencies
COPY requirements.txt ${LAMBDA_TASK_ROOT}/
RUN pip install --no-cache-dir -r ${LAMBDA_TASK_ROOT}/requirements.txt

# Copy handler and shared prompt utilities
COPY handler.py ${LAMBDA_TASK_ROOT}/handler.py
COPY prompt_utils.py ${LAMBDA_TASK_ROOT}/prompt_utils.py

# Copy ONNX model and tokenizer files (must exist locally before docker build)
# Populated by Phase 3 ONNX export: model.onnx + tokenizer_config.json + tokenizer.json + vocab.json + merges.txt + special_tokens_map.json
COPY model_artifacts/ ${LAMBDA_TASK_ROOT}/model_artifacts/

# Mangum handler: handler.py exposes 'handler' as the Mangum(app) instance
CMD [ "handler.handler" ]
```

Key points:
- Base image: `public.ecr.aws/lambda/python:3.12` (includes Lambda RIE for local testing)
- `TRANSFORMERS_OFFLINE=1` prevents tokenizer from attempting Hub download
- `TOKENIZERS_PARALLELISM=false` prevents Rust tokenizer fork warning on Lambda
- `COPY model_artifacts/` copies ONNX model + tokenizer files into the image
- `CMD ["handler.handler"]` points to the Mangum instance in handler.py
- Docker build context is the `lambda/` directory (build with `docker build --platform linux/amd64 --provenance=false -t housing-predictor:latest lambda/`)
